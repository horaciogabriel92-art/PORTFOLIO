El Desafío

Personal Shopper UY necesitaba un eCommerce de moda que no dependiera de temas multipropósito lentos. El objetivo era lograr la máxima conversión, un rendimiento optimizado, y una experiencia de usuario excepcional especialmente para dispositivos móviles.

Stack Tecnológico

  • Backend: PHP 7.4+, WordPress (CMS), WooCommerce (motor eCommerce)
  • Frontend: HTML5, Tailwind CSS (CDN), Vanilla JavaScript
  • DevOps: Python 3 para scripts de automatización y diagnóstico remoto via REST API
  • Page Builder: Elementor (solo para páginas institucionales, no en tienda)

Funcionalidades de Conversión

Barra de Envío Gratis

Barra de progreso dinámica que calcula cuánto le falta al usuario para calificar para el envío gratuito (umbral de $3000 UYU).

Gatillos de Urgencia

Contador que indica: "X personas viendo este producto en este momento" para crear escasez percibida.

Stock en Tiempo Real

Sistema de AJAX polling que consulta el stock del servidor cada 15 segundos para evitar carritos fallidos.

Sticky Cart (Móvil)

Botón de compra flotante permanente en móviles al hacer scroll, asegurando que la acción principal nunca se pierda.

Incentivo Transferencia -10%

Avisos destacados mostrando precio con 10% de descuento automático al pagar mediante transferencia bancaria.

Checkout Adaptado a LATAM

Creación de una Página de Agradecimiento Personalizada para pagos manuales (transferencia BACS). Muestra claramente los datos del Banco (Itaú) e integra un Botón flotante de WhatsApp que lleva al usuario directo al chat para enviar su comprobante.

Solución a Problemas Técnicos

Desafío: Rendimiento vs. Elementor

Elementor añade demasiado DOM y scripts pesados, destrozando los Core Web Vitals en páginas de producto.

wp_enqueue_scripts ps_dequeue_elementor_on_shop

Filtro que desactiva forzosamente Elementor en páginas nativas de WooCommerce. Elementor solo carga en landings estáticas.

Desafío: Deploy Remoto

El cliente no disponía de flujos CI/CD y el editor de temas estaba deshabilitado por seguridad.

deploy_wp.py + remote_diagnostics.py

Scripts Python que interactúan con la REST API de WordPress mediante Application Passwords para diagnóstico remoto.

Desafío: Cachés Agresivas

Los cambios CSS no se visualizaban por cachés de servidor (LiteSpeed/Redis) y de navegador.

time() como versión en wp_enqueue_style

Cache Busting dinámico que fuerza al navegador a descargar el nuevo CSS. Hooks wc_get_template con prioridad alta para saltar transients.
